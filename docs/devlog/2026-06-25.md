## Sprint 5.6 Wakeword Live Microphone Readiness Follow-Up

Record date: `2026-06-25`.

Capability:

- Extended `wakeword-service` beyond simulated events with a `WAKE_WORD_MODE`
  switch for `mock` and `live`.
- Selected Picovoice Porcupine as the formal Mandarin wake-word provider for
  reliability-first customer demo use.
- Fixed the formal wake phrase as `小慧你好`.
- Added live microphone loop configuration for Porcupine detection: Picovoice
  AccessKey, custom Linux `.ppn` keyword file, Mandarin `.pv` language model,
  sensitivity, and microphone device index.
- Added threshold/cooldown gating helpers so live detection emits
  `wake.detected` only after Porcupine fires and cooldown controls pass.

  call `/simulate-wake`; it verifies Porcupine live readiness and waits for a
  real `wake.detected` event.

  `WAKE_WORD_PHRASE=小慧你好`, `PICOVOICE_ACCESS_KEY`,

  listener readiness, microphone errors, and thread-safe event broadcast.

  `docs/dev/LOCAL_DEV.md`, and `docs/ops/LIVE_PROVIDER_RUNBOOK.md` document
  mock and live microphone operations.
- The local Mandarin `.pv` model is present at
  `.local/models/picovoice/porcupine_params_zh.pv`.
- Verification covered Python syntax, wakeword sidecar unit tests, mock
  `smoke:wakeword`, Avatar controller tests, Node syntax for wakeword smoke
  scripts, and `git diff --check`.
