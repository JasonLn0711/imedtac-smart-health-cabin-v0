# 2026-07-10 Devlog

## Sprint Context

- Sprint: Sprint 5 live acceptance hardening.
- Planned finish line: real local ASR, Gemma 4 E4B, BreezyVoice default voice,
  provider acceptance status, live-check scripts, and five-run demo evidence.

## Actual Outcome

- Completed: provider status now exposes live acceptance eligibility for ASR,
  LLM, TTS, and Redpanda.
- Completed: ASR, LLM, and TTS provider status now exposes GPU-only acceptance
  controls: `computeBackend=gpu`, `gpuRequired=true`, `cpuOffload=false`, and
  `cpuFallbackAllowed=false`.
- Completed: LLM adapter accepts `VLLM_BASE_URL` / `VLLM_MODEL` and the current
  local OpenAI-compatible Gemma 4 E4B endpoint.
- Completed: ASR adapter accepts `ASR_LANGUAGE`; existing path config supports
  the current `http://localhost:8001/asr` service.
- Completed: TTS adapter and contract keep BreezyVoice on `default` voice and
  reject customized voice configuration or request fields.
- Completed: `live:check`, `smoke:api`, and `smoke:redpanda` scripts were added.
- Completed: `@shc/voice-agent-server` was added as a runnable process that
  proxies agent-turn routes and checks strict selected-LLM readiness.
- Completed: `smoke:sprint5-live-demo` was added as a repeatable five-run live
  demo script covering admin publish, active questionnaire load, vLLM guidance,
  BreezyVoice question audio, ASR answer transcription, deterministic answer
  mapping, questionnaire submit, safety route, and public report retrieval.
- Completed: strict local vLLM Gemma 4 E4B runs at `http://localhost:8000/v1`
  without CPU offload. `VLLM_USE_FLASHINFER_SAMPLER=0` is the current GPU-only
  workaround for missing system CUDA toolkit / FlashInfer sampler JIT.
- Completed after provider comparison: native Ollama is the preferred Gemma 4
  E4B runtime for the LLM slot because it can disable thinking with
  `think:false`, returns usable short Traditional Chinese guidance, is
  lower-latency, lower-VRAM, and avoids the local vLLM GGUF loader patch.

## Scope Control

- Strict Sprint 5 acceptance uses GPU-only AI model inference. CPU backend,
  CPU fallback, or CPU offload is not acceptance eligible.
- LLM output quality is controlled by the deterministic guidance fallback guard.
  Native Ollama is used with `think:false`; the API only uses generated guidance
  when it contains usable Traditional Chinese, and answer mapping remains
  deterministic and confirmation gated.
- BreezyVoice remains default-voice only. Customized voice fields and configured
  custom voice IDs remain rejected.
