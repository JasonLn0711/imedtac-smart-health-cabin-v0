## 原始專家建議全文

結論：**單純「一台機器租三年」很難合理回到 150 萬；但「智慧健康量測服務導入＋軟硬體租用＋維運＋成效資料服務」可以設計到 150 萬，而且可以不走資本門。**
關鍵不是把 15 萬／年的租金硬灌高，而是把採購標的從「租設備」改成「租用一套持續營運的智慧健康服務」。

但要先講清楚：**不能用假拆標、假租賃、假小額採購。** 這會被主計、採購或審計打回來。

深耕計畫本身有資本門上限。計畫經費原則明列「資本門」原則不得超過 30%，但「業務費」底下可以有租金、權利使用費、設備使用服務費；租金項目也包含機器設備、資訊軟硬體等租用。相反地，如果是購置單價超過 1 萬元、耐用年限 2 年以上的軟硬體設備，就容易落入設備費／資本門。([HTSprout][1])

所以你們的可行路徑是這個：

**不要賣機器。不要移轉設備所有權。不要移轉原始碼與 IP。改成服務案。**

標案名稱可以設計成：

「健康台灣深耕計畫－智慧健康量測站軟硬體租用暨導入維運服務案」

或更穩一點：

「智慧健康量測服務導入、軟硬體租用暨成效資料彙整服務案」

這樣 150 萬買的不是一台機器，而是：

設備使用權、軟體使用權、流程導入、後端整合、API 串接、資料結構、QR 報告、四模組服務、教育訓練、維運 SLA、資安與隱私文件、成效報告。

這才有辦法合理化價格。

政府採購上也要小心。台灣目前小額採購門檻是 15 萬元，公告金額是 150 萬元；採購作業指引也明確要求，已知同一需求應合併辦理，不應為了規避較嚴格程序而拆成很多小案。經常性年度採購也應估算年度需求、必要時採開口契約或合併處理。([臺北市政府採購申訴與調解業務資訊網][2])

所以老師說「一年不要超過 15 萬」其實是**小額採購友善策略**，不是商業模式天花板。若院方堅持每年只用一張 15 萬以下小額採購，那你們就不可能用同一需求合法回收 150 萬。這點要直接面對。

你們可以採三種合法架構。

第一種，也是我最建議的：**單一混合型服務案，總價 149 萬或 150 萬。**

如果醫院想避開公告金額以上程序，實務上可用 **149 萬含稅** 比 150 萬整數更安全；如果院方可以走正式程序，就用 150 萬。這不是資本門，因為院方取得的是服務與使用權，不是買斷設備。

可以這樣拆：

| 工作包 | 內容 | 金額 |
| --- | ---: | ---: |
| 專案管理、需求凍結、驗收規劃 | 會議、範圍控管、驗收文件 | 90,000 |
| API／資料架構／系統介面設計 | session、schema、API contract、資料流 | 180,000 |
| 後端與報告模組 | QR、staff review summary、報告產生、紀錄管理 | 320,000 |
| 四模組導入 | 視力、聽力、Vital、問卷／Avatar 互動流程 | 240,000 |
| 雲端／邊緣部署與整合測試 | Render/on-prem 選項、環境設定、E2E 測試 | 220,000 |
| QA／資安／隱私／文件 | 測試案例、錯誤處理、權限、SOP | 160,000 |
| 設備租用與現場佈署 | 健康量測站使用權、現場安裝、調校 | 180,000 |
| 教育訓練與維運報告 | 教學、月報、SLA、異常處理 | 100,000 |
| **合計** |  | **1,490,000** |

這個結構的邏輯是：**硬體租金只是其中一部分，真正高成本是導入、整合、維運與軟體使用權。**

這也符合成本現實。TISSA 115 年資訊服務委外經費估算原則列出的資訊服務人力行情，專案經理、系統分析、軟體開發、雲端／邊緣整合、QA 等月費都在十幾萬到二十幾萬等級；一個 45 萬案子根本不可能涵蓋完整客製導入與維運。([Tissa][3])

第二種：**三年 managed service，總價 150 萬，但不要假裝每年都是小額。**

例如：

| 年度 | 內容 | 金額 |
| --- | ---: | ---: |
| 第一年 | 導入、整合、初始佈署、驗收、教育訓練 | 900,000 |
| 第二年 | 維運、版本更新、資料月報、SLA | 300,000 |
| 第三年 | 維運、版本更新、成效報告、續約支援 | 300,000 |
| **三年合計** |  | **1,500,000** |

這種做法比較誠實：第一年貴，是因為建置與導入成本集中在第一年；第二、三年是維運與使用權。付款可以談季付或月付，但採購金額仍應看整體契約，不應刻意拆成三個不相干的小額案。

第三種：**多點租賃＋中央平台服務。**

如果院方或北市聯醫其實不是只放一台，而是華山、信義、忠孝或多個健康服務據點，那租賃模式就更合理。

例如：

| 項目 | 金額 |
| --- | ---: |
| 3 台健康量測站，每台每年 149,000，租 3 年 | 1,341,000 |
| 中央平台開通、資料彙整、報表與後台服務 | 159,000 |
| **合計** | **1,500,000** |

這比「一台機器租三年 150 萬」更合理，因為價格來自數量與平台服務，而不是單台租金暴增。

你要求「台灣市場有沒有人這樣做過」，答案是有，而且不只一種。

苗栗縣政府衛生局就有「智慧科技整合社區健康照護設備租用案」，採購金額與決標金額都是 300 萬，得標廠商為大同醫護，標的就是健康照護設備租用，且資料顯示經費來源與國衛院補助有關。這直接證明：健康科技設備租用案可以是百萬級，不必被 15 萬小額採購綁死。([Miaoli County Government][4])

另外，原住民族委員會也有「原住民族長者 AI 健康促進系統建置及設備租賃」類型的案子，內容包含平台服務、教育訓練、語音辨識、智慧平板、智慧音箱，以及 AIO 生理量測設備租賃。這比較接近「系統服務＋設備租賃＋健康促進場域」的混合模式。

醫院軟體租賃也有更高金額的例子。採購資料彙整可查到亞東醫院「健康台灣深耕計畫－智慧藥品檢核 API 臨床決策支援系統 Medi-Span 租賃採購案」，決標金額約 1,920 萬。這說明醫療院所不是不能用租賃／使用權模式採購高價軟體，重點是標的價值、服務範圍與採購程序要站得住腳。([PCC][5])

你們現在真正要守住的底線是：

**45 萬只能做基本租賃與簡單展示，不應承諾客製開發。**

可以對老師與慧誠這樣講：

本案若採「單台設備租賃」模式，每年 15 萬、三年 45 萬，僅能涵蓋基礎設備使用、既有軟體展示與有限度維護，無法涵蓋四模組導入、後端服務、API 串接、資料結構設計、QR 報告、教育訓練、資安文件、維運 SLA 與成效資料彙整。

若院方需要的是可實際落地的智慧健康量測服務，建議採「智慧健康量測站軟硬體租用暨導入維運服務案」設計。本案不移轉設備所有權，不移轉軟體原始碼與 IP，院方取得租期內的設備使用權、軟體使用權、導入服務、維運支援與成效資料服務。此方式可列為租金、權利使用費、設備使用服務費與專業服務費的混合型業務費，而非單純設備購置。

最務實的建議：**不要再把 150 萬包裝成「租一台機器」。那會很難看，也很難守。要改成「1.49M 的智慧健康服務導入案」；硬體租賃只是其中一個子項。**

若院方堅持只願意走 15 萬以下小額採購，那你們應該降 scope：只提供既有產品租用，不做客製、不做完整後端、不做額外整合。否則就是用你們的人力成本補貼醫院專案，商業上不該接。

## Accepted Decision

Jason wants to adopt the first structure:

```text
單一混合型服務案，總價 NTD 1,490,000 或 NTD 1,500,000。
```

The adopted v4 interpretation should treat the hospital as receiving service
and use rights, not equipment ownership, source code, or IP transfer. The
`NTD 450,000` path remains a reduced-scope fallback for basic rental and simple
demonstration only.
