## 一、報價依據與市場合理性

本案採「軟硬體租用暨導入維運服務」設計。健康台灣深耕計畫第一階段申請
作業須知列明 `114-118` 年度計畫範疇包含「導入智慧科技醫療」，並在經費
使用原則中列有業務費、租金、權利使用費與設備使用服務費等科目；租金可
包含機器設備與資訊軟硬體租用。這支持本案以使用權、服務導入與維運方式
承接。來源：健康台灣深耕計畫 PDF。

政府採購門檻方面，自 `112 年 1 月 1 日` 起，公告金額為新臺幣 `150 萬元`，
中央機關小額採購金額為新臺幣 `15 萬元以下`。本案 `149 萬含稅` 的定位是
未達公告金額的完整服務案。

市場類比方面，陳仁德醫院公告「健康台灣深耕計畫 114 年－115 年智慧科技
病房 IOT 整合建置租賃服務採購案」，預算金額 `1,499,990` 元，採購金額級距
標示為未達公告金額，品項名稱為「智慧科技病房IOT整合建置租賃服務」。這
與本案的 149 萬級、智慧醫療場域、租賃服務與整合導入邏輯相近。

苗栗縣政府「苗栗縣智慧科技整合社區健康照護設備租用案」採購金額與預算
金額為 `3,000,000` 元，標的性質為租賃，補助機關為國家衛生研究院，採公開
招標與最有利標。這支持智慧健康照護設備租用與健康科技服務可採百萬級規模。

人力成本估算採中華民國資訊軟體服務商業同業公會 `115` 年資訊服務委外經費
估算原則。TISSA 說明營業稅依現行稅率 `5%` 估算，投入人月可依專案範圍、
作業方式、複雜度、服務特性與實際需求估算；第一類適用於單一應用作業或
總人月在 `24` 月以下。本案估算總投入約 `5.82` 人月，採第一類人月單價作為
保守基準。

## 五、交付項目與驗收方式

### WP1 專案啟動、需求凍結、驗收規劃、產品流程、ICD 與 API/Data Contract

交付內容：

1. 專案啟動會議 1 次。
2. 需求邊界文件 1 份。
3. 工作分工表 1 份。
4. 驗收項目表 1 份。
5. 風險與啟動條件清單 1 份。
6. User journey：開始量測、問卷、Avatar、視力、聽力、Vital、報告、QR 流程。
7. State machine：not_started、in_progress、completed、failed、skipped。
8. API contract：OpenAPI / Swagger endpoint、request / response JSON schema。
9. Session contract：session_key、TTL、idempotency_key、重複送出處理。
10. Error contract：錯誤碼、前端顯示文字、重試邏輯。
11. Data contract：問卷答案、測驗結果、報告摘要、QR token、audit log。
12. Mock data：供慧誠前端開發與聯調使用。

驗收方式：院方、慧誠與我方確認需求邊界、交付清單、驗收條件與 API / Data Contract；前端可依 mock data 進行開發。

### WP2 後端 API、Session 與模組狀態服務

交付內容：

1. Session 建立、查詢、更新與結束 API。
2. 模組狀態 API。
3. 問卷、Vital、視力、聽力、Avatar 結果接收 API。
4. idempotency 處理。
5. 基本錯誤處理。
6. 基本權限與存取控制。

驗收方式：指定測試案例可成功建立 session、更新模組狀態、接收結果並產生 report_id。

### WP3 問卷、報告、QR 與資料模型

交付內容：

1. 問卷 schema 與版本管理。
2. 問卷答案紀錄。
3. 結果摘要 mapping。
4. staff_review_summary 產生。
5. QR report token。
6. public report 頁面資料 API。
7. audit log。

驗收方式：完成一筆完整測試流程後，可產生 report_id，並可透過 QR 開啟指定報告頁。

### WP4 AI / Avatar / 視力 / 聽力模組整合邏輯

交付內容：

1. Avatar / AI adapter。
2. Avatar 狀態交接與 log 邊界。
3. 視力結果接收與摘要。
4. 聽力結果接收與摘要。
5. 非診斷、非分級、非醫囑之結果文字。
6. 報告整合規則。

驗收方式：Avatar、視力、聽力結果可進入同一份 staff review summary；結果文字維持健康檢測摘要、現場人員覆核與後續諮詢建議範圍。

### WP5 雲端 / 地端部署、慧誠前端聯調與現場導入

交付內容：

1. 測試環境 1 組。
2. 正式環境 1 組。
3. 前後端聯調支援 2 次。
4. 現場導入支援 1 次。
5. E2E 測試資料。
6. 環境設定文件。

驗收方式：慧誠前端可呼叫正式 API；現場設備可完成至少 3 筆完整測試流程。

### WP6 QA、資安、個資邊界與文件

交付內容：

1. E2E 測試案例。
2. 錯誤處理測試。
3. 基本資安檢查。
4. 個資資料流邊界說明。
5. 操作 SOP。
6. 管理端簡易操作文件。
7. 驗收報告。

驗收方式：測試案例通過率達 95% 以上；重大阻斷性錯誤完成修正。

### WP7 健康量測站設備使用服務與軟體使用權

交付內容：

1. 健康量測站一套於履約期間內之使用服務。
2. 既有軟體模組使用權。
3. 基本裝機與物流支援。
4. 基本保固協調。
5. 設備使用與操作說明。

權利與啟動條件：

1. 本項為設備使用服務；我方保留設備所有權。
2. 硬體買斷與設備所有權移轉可由院方另案採購或改列資本門評估。
3. 若硬體合作方正式租賃成本高於本報價估算，超出部分可另議或同步調整功能範圍。

### WP8 12 個月維運支援、月報與成效資料服務

交付內容：

1. 12 個月遠端維運支援。
2. 每月使用狀況摘要。
3. 每月異常紀錄與改善建議。
4. 每季成效資料彙整。
5. 操作教育訓練 1 場。
6. 院方回饋會議 2 次。
7. 小幅文字、流程與參數調整。

SLA 建議：

1. 一般問題：3 個工作日內回覆。
2. 影響測試流程之問題：1 個工作日內回覆。
3. 系統阻斷性問題：4 個工作小時內初步回覆。
4. 24/7 駐點服務可作為另案啟動項目。

## 八、對主計 / 採購的建議說法

本案標的是智慧健康量測服務之導入、租用、維運與成效資料服務。院方於履約
期間取得設備使用服務、軟體使用權、導入服務、維運支援與成效資料彙整成果；
我方保留設備所有權、軟體原始碼、模型、API 架構、資料結構與 know-how。

本案可對應健康台灣深耕計畫中「導入智慧科技醫療」與健康照護流程優化方向；
經費科目建議由院方主計依實際採購文件認定為資訊軟硬體租金、權利使用費、
設備使用服務費及相關資訊服務勞務。採購總額為 `149 萬含稅` 時，可作為
未達公告金額之完整服務案處理。

## 九、議價策略與範圍調整

本案標準報價為 `149 萬含稅`。議價時以 scope 調整方式維持交付品質：

| 降價目標 | 範圍調整項目 |
| ---: | --- |
| 140 萬含稅 | 減少一次回饋會議、月報改為季報、前後端聯調由 2 次調整為 1 次 |
| 130 萬含稅 | 現場導入改為遠端導入；教育訓練改為線上 |
| 120 萬含稅 | AI / Avatar adapter 另案啟動；主案保留問卷、Vital、報告與 QR |
| 100 萬含稅以下 | 轉為展示型 MVP / PoC，聚焦展示租用、基礎流程與短期驗證 |

本案的報價核心是：`149 萬` 對應第一年度完整導入與營運服務。三年全包方案
可另行設計三年 managed service 合約，讓第二、三年度維運、版本更新、成效
報告與續約支援有清楚服務費。
