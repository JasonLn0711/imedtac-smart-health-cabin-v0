# 智慧健康量測站合作案報價決策版

## 決策摘要

本合作案可用三個報價版本呈現，讓合作方依預算結構、硬體採購責任與軟體
交付深度選擇最適合的合作路徑。

| 版本 | 名稱 | 報價 | 決策用途 |
| --- | --- | ---: | --- |
| A | 智慧健康量測站兩台部署暨 NYCU 四模組 MVP 精簡版 | `NTD 1,500,000` | 適合總預算需要同時呈現兩台硬體部署與四模組 MVP demo 的情境。 |
| B | NYCU 智慧健康量測站四模組軟體與系統整合底線版 | `NTD 1,060,000` | 適合慧誠或院方負責硬體，NYCU 提供四模組軟體與整合交付的基本版本。 |
| C | NYCU 智慧健康量測站四模組軟體與系統整合建議版 | `NTD 1,500,000` | 適合追求較完整交付品質、驗收支援與營運導入準備的合作版本。 |

建議決策順序：

1. 先確認量測站硬體由慧誠或院方採購，或納入同一總預算。
2. 若硬體另行處理，建議採版本 C 作為完整軟體整合提案，版本 B 作為合作底線。
3. 若總預算固定為 `NTD 1,500,000` 且需涵蓋兩台量測站，採版本 A，並以 MVP 精簡交付完成第一階段展示與驗收。

## 合作案定位

本案以開放式智慧健康量測站為主體，NYCU 提供四個使用者面向模組的軟體與
系統整合能力：

| 模組 | 交付價值 |
| --- | --- |
| 問卷模組 | 建立版本化問卷、答案紀錄、基礎分數/旗標與報告摘要。 |
| Avatar 整合模組 | 串接既有 Avatar vendor，提供問卷導引、狀態交接與觸控接續流程。 |
| 視力模組 | 提供保守自我篩檢流程、距離/螢幕提示、結果文字與報告整合。 |
| 聽力模組 | 提供保守自我篩檢流程、環境與設備提示、結果文字與報告整合。 |

跨模組能力包含 kiosk 前台、session/data/API contract、共用報告、QR Code、
JSON/API 匯出、部署設定、驗收腳本、操作文件與教育訓練。這些能力支撐第一
階段的展示、驗收與後續營運擴充。

本案採 115 年資訊服務委外經費估算原則作為人月估算基準。第一類資訊服務案

`NTD 226,912`、測試人員 `NTD 163,261`。

適合用於 MVP 精簡交付。版本 B 的 `NTD 1,060,000` 約可支撐 `5.5` 人月，
可作為四模組軟體與系統整合的底線版本。

### 健康量測站硬體類比法

智慧健康量測站通常整合連網硬體、軟體平台、APP 與生理量測設備，可涵蓋

`NTD 300,000/台` 作為第一階段硬體暫估，適合作為總預算配置試算；正式報價
階段建議由慧誠提供實際硬體價格，以便鎖定最終採購與交付內容。

## 建議決策

建議以版本 C 作為完整合作提案，以版本 B 作為硬體另計時的基本合作底線。
若合作方希望在 `NTD 1,500,000` 內同時完成兩台硬體與四模組展示，版本 A
提供清楚、可執行的 MVP 精簡交付路徑。

正式報價前建議完成三項確認：

1. 慧誠提供健康量測站正式硬體規格與價格。
2. Avatar vendor 提供 API / iframe / web component / SDK 型態與商業授權條件。
3. 合作方確認第一階段採 demo / MVP 驗收、正式營運導入，或進一步 HIS/EMR 整合路徑。
